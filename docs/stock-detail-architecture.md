📘 BullSignalsAI — Stock Detail Screen Architecture

Purpose
This document defines the final architecture, data flow, and optimization strategy for the Stock Detail Screen in BullSignalsAI.
It is the single source of truth before implementation.

1. Goals & Principles
🎯 Goals

Sub-second perceived load time

Accurate, near-real-time data

Zero heavy computation on frontend

Minimal API and AI token usage

Scalable to millions of users

Clear separation of concerns

🚫 Non-Negotiables

No direct external API calls from UI

No Pandas / NumPy work in request path

No Grok calls from frontend

No recomputation per user tap

2. High-Level Data Flow
UI Request Path (Fast & Lightweight)
UI (React Native)
   ↓
Service Layer (JS)
   ↓
FastAPI (main.py)
   ↓
Firestore (read-only)

Data Production Path (Async)
Cron Jobs
   ↓
APIs + Models + AI (BullBrain + Grok)
   ↓
Firestore (precomputed, cached)


The UI only reads Firestore-backed endpoints
All computation happens ahead of time

4. Section Architecture Details
4.1 Header (Price & Range)

Data

Price, % change

Day range, open, prev close

Volume (short format)

Timestamp (“As of”)

Source

Firestore → stock_realtime/{symbol}


Updated by

Cron every 30–60 seconds

4.2 AI Hybrid Signal (Critical)

Includes

BUY / HOLD / SELL

Confidence %

Upside probability

BullBrain v2 output

Human narrative (“why”)

Source

Firestore → stock_ai_snapshot/{symbol}


Computed by

BullBrain v2 (48 features)

Technical snapshot

Hybrid blending logic

4.3 Smart Pattern Alert ⭐

Includes

Pattern name

Bias (bull/bear/neutral)

Win rate

Explanation

Historical forward returns

Sample past dates

Source

Firestore → stock_smart_pattern/{symbol}


Engine

Existing scan_smart_pattern_history()

Uses ~1 year daily candles

Important

Never computed in request path

Never triggered by user tap

4.4 Outlook (Short / Medium / Long)

Source

Firestore → stock_ai_snapshot/{symbol}


Generated by

Grok (structured prompt)

4.5 Technical Snapshot (Humanized)

Includes

Trend

Momentum

Volatility

Volume behavior

Plain-English explanations

Source

Firestore → stock_technical_snapshot/{symbol}

4.6 News & Market Sentiment

Source

Firestore → stock_news/{symbol}


Updated by

newsdata.io cron

Deduplicated

Cached aggressively
